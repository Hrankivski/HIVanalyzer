## 📌 Про проєкт (Мета)
**HIVanalyzer** — це науково-дослідницький інструмент та симуляційна екосистема, розроблена для створення високоточних синтетичних наборів даних (synthetic datasets) про термодинаміку приміщень та якість повітря. 

Головна мета проєкту — генерувати реалістичні фізичні дані роботи систем ОВКПа (опалення, вентиляція, кондиціювання повітря) з використанням потужного рушія **EnergyPlus**. Ці дані враховують теплову інерцію стін, тепловиділення від обладнання, динаміку CO₂, а також викиди специфічних забруднювачів (наприклад, частинок від паяння або 3D-друку). Надалі згенерований датасет буде використаний для **навчання моделей машинного навчання (ML)**, які керуватимуть "розумною" системою оптимізації рекуператорів та вентиляції (Demand Controlled Ventilation).

## 🎯 Очікуваний фінальний вигляд 
У своєму повністю завершеному стані проєкт являтиме собою **автоматизований AI-пайплайн**:
1. **Інструмент моделювання (Streamlit Dashboard):** Користувач задає параметри приміщення, визначає матеріали стін, розклади активності людей та потужність обладнання.
2. **Симуляційний рушій:** Автоматичний генератор (IDF Bridge) інтегрує ці параметри з рушієм EnergyPlus. Система випадковим чином варіює умови (погоду, завантаженість) для масової генерації batch-симуляцій.
3. **ML-Агент Оптимізатор:** Натренована нейромережа, яка на основі історичних даних (створених симулятором) у реальному часі рекомендує оптимальний відсоток роботи рекуператора/вентиляторів для забезпечення балансу між ідеальним станом повітря (низький CO₂, відсутність пилу) та мінімальним споживанням енергії.

## 🚀 Поточний етап розробки
Наразі проєкт знаходиться на **Фазі 9.5 "Master Sync & Physical Fidelity"** (Глибока валідація даних).
Ми завершили створення пайплайну генерації даних і сфокусовані на абсолютній фізичній достовірності моделі перед стартом ML-навчання.

**Що вже реалізовано і працює:**
- [x] **Параметрична генерація геометрії:** Код генерує IDF-файли для EnergyPlus "на льоту". Формуються L-подібні приміщення, вікна, зони.
- [x] **True DCV (Demand Controlled Ventilation):** Вентиляційні вентилятори та рекуператори (ERV) не використовують жорсткі розклади, а динамічно і плавно керуються через вбудовані програми керування **EMS (Energy Management System)** залежно від рівня CO₂ у зоні.
- [x] **Комплексне енергомоделювання:** Моделюється ідеальне опалення/охолодження без нічних спадів (setbacks) для ідеальної кореляції енергії з температурним градієнтом.
- [x] **Генерація випадкових подій (Забруднення):** Додано систему симуляції раптових викидів Generic Contaminants (паяння/3D-друк) з випадковою частотою, що виступає потужним стрес-тестом для вентиляції.
- [x] **Модуль валідації (`validator.py`):** Вбудований набір unit-тестів симуляції (перевірка теплової інерції Thermal Decay, адекватності очищення повітря, ефективності теплообмінників та виявлення неможливих величин).
- [x] **Batch Pipeline:** Інструмент вміє масово проганяти симуляції та складати результат у готовий для AI файл `training_data.csv`.

**Наступний крок:**
- Перехід до етапу Data Science: побудова архітектури ML-моделі (RL-агент або Regression), підготовка пайплайну навчання (PyTorch/Scikit-Learn) на створеному датасеті та розробка логіки inference (застосування моделі) у Streamlit застосунку.
