# Маніфест Проєкту: "Recuperator Optimizer"

## 1. Загальна інформація
- **Тема дипломної роботи**: Підбір рекуператора методами машинного навчання на основі цифрового двійника приміщення.
- **Студент**: Дмитро Гранківський (4-й курс, спеціальність 121 «Інженерія програмного забезпечення»).
- **Мета**: Створення програмного комплексу для точного моделювання мікроклімату та автоматизованого вибору оптимальної моделі рекуператора.

## 2. Технологічний стек
- **Мова**: Python 3.13.
- **GUI**: Streamlit (використовується остання версія з CSS-хаками для прозорості).
- **Візуалізація**: Plotly (обрано замість st-canvas для забезпечення інженерної точності та стабільності координат).
- **Обробка даних**: Pandas (зберігання стану об'єктів у DataFrame).

## 3. Архітектура системи
Проєкт розділено на два основні модулі для дотримання принципів чистого коду:
- `main.py`: Точка входу. Керує інтерфейсом (Tabs), бічною панеллю (Sidebar) та станом сесії (`st.session_state`). Містить логіку взаємодії користувача з параметрами.
- `designer.py`: Інженерне ядро. Відповідає за математичну модель приміщення, генерацію Plotly-креслення та розрахунок геометрії об'єктів.

## 4. Поточний функціонал GUI
| Функція | Опис |
| :--- | :--- |
| **Параметричне моделювання** | Ввід L, W, H через `number_input` з автоматичним перерахунком об'єму. |
| **BIM-конструктор** | Додавання вікон та рекуператорів як об'єктів з унікальними ID. |
| **Логіка позиціонування** | Реалізовано вільні координати $X, Y$ з функцією «швидкої прив'язки» до 4-х стін (Пд, Пн, Зх, Сх). |
| **Динамічна орієнтація** | Автоматичний або ручний розворот об'єктів (горизонтально/вертикально). |
| **Persistence** | Система експорту всього стану проєкту (геометрія + об'єкти + налаштування) у JSON формат. |

## 5. Ключові технічні рішення (Context)
- **Coordinate System**: Ми використовуємо метричну систему координат. Масштаб у Plotly зафіксовано як 1:1, що дозволяє передавати дані в EnergyPlus без додаткових трансформацій.
- **State Management**: Кожен об'єкт на плані синхронізований з `st.session_state` через унікальні ключі (наприклад, `x_0`, `y_0`). Це дозволяє кнопкам «прилипання» керувати слайдерами в реальному часі.
- **Physics Ready**: У моделі вже закладено параметри товщини стін, матеріалів та площі скління (Ширина $\times$ Висота вікна), що необхідно для розрахунку теплового балансу.

## 6. Поточний статус та плани
- **Виконано**: GUI-прототип (Версія 0.1.3 Stable).
- **Наступний крок**: Розробка модуля інтеграції з **EnergyPlus** для генерації IDF-файлів на основі JSON-маніфесту проєкту та впровадження **ML-моделі** для аналізу ефективності рекуперації.
